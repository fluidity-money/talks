Pros Cons
  • Fast and peer to peer, can support different configurations with multiple parties with liveliness requirement
  • Highly efficient application-specific form of compression (usually the penultimate state within the application)
  • Can involve immediate finality of funds
  • Peer to peer implies some degree of privacy
  • Can require locking funds up for a exit period
  • Requires liveliness for data consistency for each user unless a hub is used to route everything
  • Ecosystem not developed like sidechains and rollups
  • User must call the contract that ends the peer to peer interaction on-chain - and the fee to service the hub provider if that was used
  • Mainstream dApps with no special solution aren't supported with the current solutions
  • Any kind of multiplayer/multiple party interaction only possible if incentives align
